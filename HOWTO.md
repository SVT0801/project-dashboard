# Как запустить расширение

## 1. Запуск в режиме разработки

### Вариант 1: С помощью F5
1. Откройте проект в VS Code
2. Нажмите `F5` или выберите `Run > Start Debugging`
3. Откроется новое окно VS Code с активированным расширением
4. В левой панели (Activity Bar) найдите иконку с сеткой - это Project Dashboard

### Вариант 2: Через командную строку
```bash
# Компиляция в watch-режиме (автоматическая перекомпиляция при изменениях)
npm run watch

# В другом терминале запустите Extension Host
code --extensionDevelopmentPath=/Users/svt/Library/Mobile\ Documents/com~apple~CloudDocs/VS_code\ PROJECTS/2026\ SVT/\[002\]\ VS-CODE\ -\ create\ dashbord\ plugin
```

## 2. Тестирование функций

### Добавление проекта
1. Откройте панель Project Dashboard в левом меню
2. Нажмите кнопку "+" в заголовке панели
3. Заполните форму:
   - Название проекта
   - Описание (опционально)
   - Теги через запятую (опционально)
   - Выберите цвет

### Редактирование проекта
- Наведите курсор на карточку проекта
- Нажмите иконку карандаша ✏️
- Измените нужные поля

### Удаление проекта
- Наведите курсор на карточку проекта
- Нажмите иконку корзины 🗑️
- Подтвердите удаление

### Переключение вида
- Нажмите иконку списка в заголовке панели
- Вид переключится между карточками и таблицей

## 3. Сборка .vsix пакета (для установки)

```bash
# Установите vsce (если еще не установлен)
npm install -g @vscode/vsce

# Создайте .vsix файл
vsce package

# Установите созданный файл
code --install-extension project-dashboard-0.0.1.vsix
```

## 4. Отладка

### Просмотр логов
- В режиме разработки откройте: `Help > Toggle Developer Tools`
- Перейдите на вкладку Console для просмотра логов

### Точки останова
1. Поставьте breakpoint в src/*.ts файлах
2. Нажмите F5
3. Выполните действие, которое должно вызвать код с breakpoint
4. Отладчик остановится на точке останова

## 7. Где хранятся данные?

Проекты сохраняются в `globalState` VS Code, что означает:
- Данные сохраняются между сессиями
- Данные доступны из любой папки/workspace
- Данные хранятся локально на вашем компьютере

Путь к данным (для справки):
- macOS: `~/Library/Application Support/Code/User/globalStorage/`
