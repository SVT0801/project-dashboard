# Инструкция по установке Project Dashboard

## ✅ Расширение готово к установке!

Пакет расширения: **project-dashboard-0.0.1.vsix** (37 KB)

---

## Способ 1: Установка через VS Code UI (рекомендуется)

### Шаг 1: Откройте VS Code

### Шаг 2: Установите расширение

**Вариант A - Drag & Drop:**
1. Перетащите файл `project-dashboard-0.0.1.vsix` в окно VS Code
2. Подтвердите установку

**Вариант B - Через Extensions панель:**
1. Откройте Extensions (⇧⌘X / Ctrl+Shift+X)
2. Нажмите на `...` (три точки) в правом верхнем углу
3. Выберите **"Install from VSIX..."**
4. Найдите и выберите файл `project-dashboard-0.0.1.vsix`
5. Нажмите "Install"

### Шаг 3: Перезагрузите VS Code

После установки VS Code предложит перезагрузить окно:
- Нажмите **"Reload Now"**
- Или закройте и откройте VS Code заново

### Шаг 4: Найдите расширение

После перезагрузки вы увидите:
- Новую иконку **"Projects"** в Activity Bar (левая панель)
- Кликните на неё для открытия панели проектов

---

## Способ 2: Установка через командную строку

Если у вас настроена команда `code` в терминале:

```bash
code --install-extension project-dashboard-0.0.1.vsix --force
```

Если команда `code` не найдена, настройте её:

### macOS:
1. Откройте VS Code
2. Command Palette (⇧⌘P)
3. Введите: **"Shell Command: Install 'code' command in PATH"**
4. После этого команда будет доступна

### Windows/Linux:
Обычно команда доступна по умолчанию после установки VS Code.

---

## ✨ Первый запуск

После установки:

1. **Откройте панель Projects** - кликните на иконку в Activity Bar
2. **Добавьте первый проект** - нажмите кнопку `+`
3. **Выберите папку проекта**
4. **Настройте проект**: имя, описание, теги, цвет
5. **Готово!** Проект добавлен

### Быстрые команды:

Откройте Command Palette (⇧⌘P / Ctrl+Shift+P) и введите:
- `Project Dashboard: Add Project` - добавить проект
- `Project Dashboard: Export Projects to JSON` - создать backup
- `Project Dashboard: Settings & Info` - открыть справку

---

## 💾 Backup ваших данных

### Создание резервной копии:

1. Command Palette (⇧⌘P)
2. Введите: **"Project Dashboard: Export Projects to JSON"**
3. Выберите место сохранения (рекомендуем: `~/project-dashboard-backup.json`)
4. Готово! Все проекты сохранены в JSON файл

### Восстановление из backup:

1. Command Palette (⇧⌘P)
2. Введите: **"Project Dashboard: Import Projects from JSON"**
3. Выберите файл backup
4. Подтвердите импорт
5. Дубликаты будут автоматически пропущены

**💡 Совет:** Создавайте backup перед:
- Переустановкой VS Code
- Обновлением или удалением расширения
- Переносом данных на другой компьютер

---

## 🔄 Обновление расширения

При выходе новой версии:

1. Скачайте новый .vsix файл
2. Установите через VS Code UI (см. Способ 1)
3. VS Code предложит заменить существующую версию
4. Подтвердите замену
5. Перезагрузите VS Code

**Ваши данные сохранятся!** Проекты хранятся в VS Code globalState и не удаляются при обновлении.

---

## 🗑️ Удаление расширения

Если нужно удалить расширение:

1. Extensions панель (⇧⌘X)
2. Найдите "Project Dashboard"
3. Нажмите **"Uninstall"**

**⚠️ Внимание:** При удалении через UI:
- Все данные (проекты) будут удалены
- Восстановить можно только из backup файла
- **Создайте backup перед удалением!**

### Восстановление после удаления:

1. Переустановите расширение (см. выше)
2. Импортируйте backup (Command Palette → Import Projects from JSON)
3. Все проекты восстановлены!

---

## 📂 Где хранятся данные

### VS Code globalState:
```
~/Library/Application Support/Code/User/globalStorage/state.vscdb (macOS)
```

В этой базе хранятся все проекты. База управляется VS Code автоматически.

### Backup файлы:
Вы можете сохранить backup в любое место:
- `~/project-dashboard-backup.json` (домашняя папка)
- `~/Dropbox/Backups/` (облако для синхронизации)
- `~/Documents/` (документы)

Backup файл - это обычный JSON, который можно редактировать вручную.

## 🆘 Устранение проблем

### Расширение не появляется после установки
1. Перезагрузите VS Code полностью (Quit → Open)
2. Проверьте Extensions панель - должно быть "Project Dashboard"
3. Проверьте Activity Bar - должна быть иконка Projects

### Потерялись данные
1. Расширение было удалено? → Импортируйте backup
2. Данные есть, но не отображаются? → Перезагрузите VS Code
3. Нет backup? → Данные хранятся в globalState, попробуйте восстановить через Developer Tools

### Не работает команда `code`
- macOS: VS Code → Command Palette → "Shell Command: Install 'code' command in PATH"
- Или установите через UI (Способ 1)

### Ошибка при импорте backup
- Проверьте что файл валидный JSON
- Убедитесь что структура соответствует формату (массив объектов с полями: id, name, tags, color)
